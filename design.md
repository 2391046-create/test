# Student Finance Compass 디자인 계획

## 디자인 방향

**Student Finance Compass**는 해외 교환학생과 유학생이 한 손으로 빠르게 생활비 흐름을 확인하고, 결제 알림에서 추출한 지출을 자동으로 정리하며, 환율 변동에 따라 송금 타이밍을 판단할 수 있도록 돕는 모바일 금융 도우미이다. 인터페이스는 **9:16 세로 화면**, **한 손 사용**, **iOS 기본 앱에 가까운 단정한 계층 구조**를 기준으로 설계한다. 핵심 가치는 사용자가 매일 은행 앱, 카드 앱, 환율 앱을 오가며 확인해야 하는 비용을 줄이고, 장학금·비자·부모님 보고에 필요한 재정 증빙을 더 투명하게 준비하도록 돕는 것이다.

## 화면 목록

| 화면 | 주요 목적 | 핵심 콘텐츠와 기능 |
|---|---|---|
| 홈 | 오늘의 재정 상태와 추천 액션을 요약한다. | 총 보유 자산 환산액, 이번 달 지출, 미분류 결제, 환율 추천 카드, 최근 거래 목록을 표시한다. |
| 기록 | 결제 알림에서 인식된 거래를 확인하고 수정한다. | 거래 리스트, 카테고리 칩, 가맹점명, 통화, 금액, 신뢰도, 직접 입력 버튼을 제공한다. |
| 리포트 | 장학금·비자·부모님 보고용 지출 증빙을 준비한다. | 기간 필터, 카테고리별 합계, 증빙 준비 상태, 트랜잭션 해시·QR 코드 형태의 검증 정보 모형을 보여준다. |
| 환율 | 환율 시계열을 기반으로 송금 타이밍을 추천한다. | KRW→USD/EUR 송금 추천, ‘지금 보내세요’ 또는 ‘N일 대기 권장’ 액션, 추세 카드, 예상 절감액을 표시한다. |
| 설정 | 카테고리 규칙과 알림 인식 설정을 관리한다. | 자동 분류 규칙, 통화 기준, 보고서 언어, 알림 샘플 붙여넣기 도움말을 제공한다. |

## 주요 사용자 흐름

| 흐름 | 단계 | 설계 의도 |
|---|---|---|
| 결제 알림 인식 | 사용자가 카드 결제 알림 문구를 붙여넣음 → 앱이 가맹점·금액·통화를 추출함 → 키워드 기반으로 식비·교통·주거·학업 등 카테고리를 추천함 → 사용자가 필요 시 수정함 | 실제 운영에서는 OS 알림 접근 권한과 금융기관 연동이 필요하지만, MVP에서는 개인정보와 플랫폼 제약을 고려해 **알림 텍스트 수동 붙여넣기 기반 시뮬레이션**으로 구현한다. |
| 지출 검토 | 사용자가 기록 탭 진입 → 최근 거래와 카테고리 확인 → 미분류 거래 선택 → 카테고리 변경 → 홈 요약과 리포트 합계가 즉시 반영됨 | 자동화 결과를 사용자가 쉽게 검증할 수 있게 하여 증빙 신뢰도를 높인다. |
| 증빙 리포트 준비 | 사용자가 리포트 탭 진입 → 기간 선택 → 카테고리별 합계 확인 → 제출용 요약 카드 확인 | 장학금·비자 제출 전 수작업 정리를 줄이는 목적의 화면이다. |
| 환율 추천 확인 | 사용자가 환율 탭 진입 → 목적 통화 선택 → 현재 환율과 7일 추세 확인 → 앱이 ‘지금 보내세요’ 또는 ‘N일 대기 권장’을 제시 | 매일 환율을 직접 확인하지 않아도 되는 경험을 제공한다. |

## 레이아웃 원칙

홈 화면은 상단에 인사말과 총 잔액을 배치하고, 엄지손가락이 닿기 쉬운 중하단 영역에 주요 액션 카드와 최근 거래 리스트를 둔다. 기록 화면은 검색이나 복잡한 필터보다 **카테고리 칩과 신뢰도 표시**를 우선하여 사용자가 자동 분류 결과를 빠르게 검토하도록 한다. 환율 화면은 그래프보다 의사결정 문구를 먼저 보여주는 구조로 설계하며, 차트와 지표는 추천 이유를 설명하는 보조 정보로 배치한다.

## MVP 범위와 제약

초기 구현은 실제 카드사·은행 알림에 직접 접근하지 않고, 사용자가 붙여넣은 결제 알림 텍스트를 파싱하여 거래를 생성하는 방식으로 구현한다. XRPL 기반 온체인 기록은 첨부 기획에서 중요한 장기 비전이지만, 현재 앱은 서버·지갑·실거래 연동 없이 **로컬 샘플 데이터와 검증 정보 모형**으로 재정 증빙 흐름을 보여준다. 환율 추천 역시 외부 API 키 없이 샘플 시계열 데이터를 분석해 명확한 액션을 제시하는 형태로 구현한다.
